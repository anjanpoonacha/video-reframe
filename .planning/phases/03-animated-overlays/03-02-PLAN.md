  <action>
  Modify the export loop in `exportVideo()` to detect skip-range boundaries and render cross-fade transitions per D-19, D-20, D-21.

  DETECTION LOGIC:
  Before the main frame loop, pre-compute a Set of "cut entry frame indices." A cut entry is the first frame rendered AFTER a skip range ends. Algorithm:
  - Sort skipRanges by start time.
  - For each frame index i, compute t = i/fps. If the PREVIOUS logical frame (i-1) was skipped (was inside a skipRange), then frame i is a "cut entry frame."
  - Store these indices in a Set called `cutEntryFrames`.

  CROSS-FADE PARAMETERS (per D-19):
  - Duration: 0.15 seconds = ~4-5 frames at 30fps (use `const FADE_FRAMES = 4`)
  - Blend: linear alpha ramp. Frame 0 of transition = 100% previous, 0% current. Frame 4 = 0% previous, 100% current. Each intermediate frame blends proportionally.

  IMPLEMENTATION (per D-20):
  - Create a secondary OffscreenCanvas (same dimensions: encW x encH) and its 2d context called `fadeCtx`. Create this once before the loop.
  - Track `lastRenderedFrame: ImageData | null = null` — after drawing+overlaying each normal frame, save ctx.getImageData() into lastRenderedFrame.
  - When the current frame is in `cutEntryFrames` AND lastRenderedFrame exists:
    - For FADE_FRAMES iterations (j = 0 to FADE_FRAMES-1):
      - alpha = (j + 1) / (FADE_FRAMES + 1) — progress from ~0.2 to ~0.8
      - Draw the current frame (crop + overlay as normal) onto fadeCtx
      - On the main ctx: putImageData(lastRenderedFrame), then set globalAlpha = alpha, drawImage(fadeCanvas, 0, 0), reset globalAlpha = 1.0
      - Encode the blended frame as a VideoFrame (with sequential timestamp)
      - Increment encodedFrames
    - After fade frames complete, continue with normal rendering of the current real frame (which is alpha=1 of the new content).
  - For non-cut-entry frames: render normally, then store `lastRenderedFrame = ctx.getImageData(0, 0, encW, encH)`.

  IMPORTANT CONSTRAINTS:
  - The overlay function MUST still be called on cross-fade frames (the overlay continues rendering during transitions).
  - VideoFrame lifecycle: each fade frame gets its own VideoFrame created and closed in try/finally (same backpressure pattern as existing code).
  - The cross-fade adds ~4 extra encoded frames per cut. Adjust progress calculation accordingly: `totalFrames` should account for estimated transition frames (add `skipRanges.length * FADE_FRAMES` to total for progress denominator).
  - No whip-pans, no glitches, no zooms — just alpha blending (per D-21).

  The fadeCanvas and fadeCtx are created once before the loop and reused for all transitions. No per-transition allocation.
