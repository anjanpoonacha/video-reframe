<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → template create | Brand kit data (including logo data URL) flows into template construction |
| data URL → createImageBitmap | Logo string decoded as image — potential malicious SVG content |
| GSAP library → canvas render | Third-party code animates state objects drawn to canvas |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | SVG logo → ImageBitmap | mitigate | Decode via createImageBitmap (sandboxed rasterizer) — never inject SVG as DOM innerHTML or use DOMParser |
| T-02-06 | Information Disclosure | Canvas tainted by cross-origin | accept | Logo is user-uploaded from local file (same-origin data URL) — no CORS issue |
| T-02-07 | Denial of Service | GSAP timeline per-frame CPU | accept | Timeline is created once and seek() is O(1) lookup — acceptable cost per RESEARCH.md |
| T-02-SC | Tampering | npm/gsap | mitigate | Package verified in RESEARCH.md Package Legitimacy Audit — 12yr npm history, official repo |
</threat_model>
