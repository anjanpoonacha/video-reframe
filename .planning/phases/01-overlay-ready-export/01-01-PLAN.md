<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file → VideoElement | User-selected video file decoded by browser's native video decoder |
| Canvas → VideoFrame | Pixel data from canvas context becomes GPU-backed VideoFrame |
| VideoFrame → Encoder | Frame submitted to hardware encoder via WebCodecs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | VideoEncoder queue | mitigate | Backpressure gate: `encodeQueueSize > 5` check before every encode (D-03) |
| T-01-02 | Denial of Service | VideoFrame GPU memory | mitigate | try/finally frame.close() on every iteration (D-04) + safety sweep assertion (D-05) |
| T-01-03 | Denial of Service | Main thread blocking | mitigate | setTimeout yield every 5 frames maintains UI responsiveness |
| T-01-04 | Tampering | Malformed video input | accept | Browser's native video decoder handles malformed files in sandbox — no application-level mitigation needed |
</threat_model>
