Optimize the proPackTemplate's `create()` and `render()` functions in src/templates.ts:

1. **Text measurement caching (D-07):** In `create()`, after getting brandKit, pre-compute text widths:
   - Create a temporary OffscreenCanvas(1,1) and get its 2d context
   - Measure `brandKit.channelName` width at the font used in the lower-third (600 weight, 18px baseline scaled)
   - Measure `brandKit.channelHandle` width at the handle font (500 weight, 14px baseline scaled)
   - Store these as closure variables `channelNameWidth` and `channelHandleWidth`
   - The render function can reference these cached widths instead of calling measureText per frame
   - Note: Currently templates.ts does NOT call measureText per-frame (uses fillText directly), but caching the width enables future use for truncation/layout decisions and satisfies D-07's intent

2. **Batched ctx.save()/restore() (D-08):** The Pro Pack render function currently has ONE save/restore pair wrapping the entire render. This already satisfies D-08. Verify the lowerThirdTemplate also follows this pattern. If it has nested save/restore pairs beyond the logo clip (which legitimately needs its own pair), consolidate them.

3. **Skip overlay when invisible (D-16):** At the TOP of the Pro Pack `render` function, add an early-return check:
   - The Pro Pack has: intro (0–2.5s), lower-third (2.5–6.3s), watermark (2.5s–end)
   - The watermark is always active after 2.5s, so the overlay is ALWAYS visible after 2.5s when brandKit.logo exists or default monogram is used
   - The only scenario where full skip is possible: `time > 6.3` AND watermark is disabled — but watermark is always on in Pro Pack (state.watermarkAlpha snaps to 1 at 2.5s)
   - Therefore: add a comment documenting this analysis. The useful optimization is WITHIN the render — skip sections that aren't active at the current time:
     - `if (time >= 2.5)` skip the intro block entirely (already has `if (time < 2.5)` guard — good)
     - `if (time >= 6.3)` skip the lower-third block entirely (already has `if (time >= 2.5 && time < 6.3)` guard — good)
   - For the watermark-only case (time > 6.3): the render already skips intro and lower-third. Only watermark draws (~one drawImage + clip). This is already ~2-5ms. Add a fast-path: if `time >= 6.3`, seek GSAP timeline ONCE (no-op since state doesn't change after 6.3s) and only draw watermark.
   - Implementation: after `tl.seek(time)`, add `if (time >= 6.3) { /* watermark-only fast path */ ctx.restore(); return; }` after drawing just the watermark. Move the watermark draw before the intro/lower-third conditionals, or duplicate the watermark draw in the fast path (simpler — watermark is 5 lines of code).

4. **Reduce canvas state changes in lower-third rendering:**
   - The lower-third section changes `globalAlpha` 7 times. This is acceptable as it's already within a single save/restore. No additional optimization needed per D-08 (one pair per major component).

5. Apply the same text-cache pattern to `lowerThirdTemplate.create()` — pre-compute channelName width in a closure variable.

<task type="auto">
  <name>Task 2: Add thermal pressure detection and frame-timing instrumentation to export loop</name>
  <files>src/export.ts</files>
  <read_first>src/export.ts, src/device-tier.ts</read_first>
  <action>
Add thermal throttle detection to the export loop in src/export.ts, per D-17:

1. **Frame timing instrumentation:** Before the main export loop, declare:
   - `let baselineFrameTime = 0`
   - `let frameCount = 0`
   - `let recentFrameTimes: number[] = []`
   - `const THERMAL_WINDOW = 20`
   - `const THERMAL_THRESHOLD = 1.8`
   - `let effectiveYieldEvery = preset.yieldEvery` (mutable — thermal detection can reduce it)

2. **Per-frame timing:** Wrap each frame's work (seek + draw + overlay + encode) with `performance.now()`:
   - `const frameStart = performance.now()` before the seek
   - `const frameTime = performance.now() - frameStart` after the encode

3. **Baseline establishment:** For the first 10 frames, compute rolling average:
   ```
   if (frameCount < 10) {
     baselineFrameTime = ((baselineFrameTime * frameCount) + frameTime) / (frameCount + 1);
   }
   frameCount++;
   ```

4. **Thermal pressure check (D-17):** After the baseline is established (frameCount >= 10):
   - Push frameTime to recentFrameTimes
   - If recentFrameTimes.length > THERMAL_WINDOW, shift
   - If recentFrameTimes.length === THERMAL_WINDOW: compute median, if `median > baselineFrameTime * THERMAL_THRESHOLD` → set `effectiveYieldEvery = Math.max(2, Math.floor(preset.yieldEvery / 2))` (yield more often)
   - Otherwise: reset `effectiveYieldEvery = preset.yieldEvery`

5. **Use effectiveYieldEvery** in the yield gate instead of the fixed `preset.yieldEvery`. The `consecutiveFrames >= effectiveYieldEvery` check (from Plan 01) now adapts dynamically.

6. **Cross-fade frames also count:** Ensure frames encoded in the cross-fade inner loop also increment `consecutiveFrames` and `frameCount`, and are timed for thermal detection.

7. Do NOT import or use PressureObserver or navigator.getBattery() — these are unavailable on the target device per RESEARCH Pitfall 3.
