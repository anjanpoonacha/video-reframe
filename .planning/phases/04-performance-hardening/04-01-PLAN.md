  <action>
Refactor src/export.ts to use adaptive performance settings:

1. Import `detectDeviceTier`, `getPerformancePreset`, `DeviceTier` from "./device-tier".

2. Add optional `deviceTier?: DeviceTier` field to `ExportConfig` interface — per RESEARCH ExportConfig pattern.

3. At the top of `exportVideo()`, detect tier and get preset:
   ```
   const tier = config.deviceTier ?? detectDeviceTier();
   const preset = getPerformancePreset(tier);
   ```

4. **Resolution adaptation (D-12, D-13):** Replace the hardcoded `outW = Math.min(1080, ...)` / `outH = Math.min(1920, srcH)` with:
   ```
   const outW = Math.min(preset.outputWidth, Math.round(srcH * (9 / 16)));
   const outH = Math.min(preset.outputHeight, srcH);
   ```
   Keep the existing `encW = outW - (outW % 2)` / `encH = outH - (outH % 2)` for even-dimension enforcement.

5. **Backpressure adaptation (D-09):** Replace both `while (encoder.encodeQueueSize > 5)` with `while (encoder.encodeQueueSize > preset.maxEncodeQueue)` — there are two occurrences (main loop line 173, cross-fade loop line 143).

6. **Yield adaptation (D-10, D-18):** Replace `if (i % 5 === 0)` yield with a `consecutiveFrames` counter:
   - Increment `consecutiveFrames` after each encoded frame (both normal and cross-fade frames)
   - Yield when `consecutiveFrames >= preset.yieldEvery || consecutiveFrames >= preset.maxConsecutive`
   - After yield: `consecutiveFrames = 0`
   - Use `setTimeout(r, preset.yieldMs)` instead of `setTimeout(r, 0)`

7. **Eliminate getImageData OOM vector (Critical — Pitfall 1 from RESEARCH):**
   Replace `lastRenderedFrame = ctx.getImageData(0, 0, encW, encH)` (line 185) with a canvas-to-canvas copy:
   - Allocate a `prevCanvas = new OffscreenCanvas(encW, encH)` and `prevCtx` ONCE before the loop (D-03, D-04)
   - After rendering each normal frame: `prevCtx.drawImage(canvas, 0, 0)` — GPU-accelerated copy, no JS allocation
   - Set a `hasPrevFrame = true` boolean (replaces null check on lastRenderedFrame)
   - In the cross-fade block: replace `ctx.putImageData(lastRenderedFrame, 0, 0)` with `ctx.drawImage(prevCanvas, 0, 0)`
   - Remove the `lastRenderedFrame: ImageData | null` variable entirely

8. Keep all other logic intact (frame seeking, skip ranges, cut entry detection, overlay injection, try/finally VideoFrame lifecycle).
